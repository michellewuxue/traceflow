# TraceFlow 产品需求文档 (PRD)

## 1. 项目概述

### 1.1 产品名称
**TraceFlow - 团队工作流与月报系统**

### 1.2 产品愿景
帮助团队更高效地记录日常工作、自动汇总月度总结，让工作有迹可循、汇报有据可依。

### 1.3 目标用户
| 用户角色 | 核心需求 |
|---------|---------|
| 团队成员（开发/设计） | 快速记录工作、自动生成月报 |
| 产品经理 | 了解项目进展、协调团队工作 |
| 项目主管 | 查看团队整体工作情况、汇总汇报 |

## 2. 核心功能

### 2.1 功能模块总览

```
TraceFlow
├─ 登录/注册
├─ 工作日志看板
├─ 撰写工作日志
├─ 个人月报
└─ 团队月报
```

### 2.2 详细功能描述

#### 2.2.1 登录与注册
**功能描述：** 用户通过邮箱密码登录或注册系统

**用户故事：**
> 作为团队成员，我想要登录系统，以便开始记录工作。

**验收标准：**
- [ ] 支持邮箱密码登录
- [ ] 支持新用户注册（自动验证）
- [ ] 支持退出登录
- [ ] 记住登录状态（刷新不丢失）

---

#### 2.2.2 工作日志看板
**功能描述：** 以横向看板形式展示团队成员的工作日志

**用户故事：**
> 作为团队成员，我想要看到所有人的日志概览，以便了解团队工作进展。

**验收标准：**
- [ ] 左侧到右侧横向展示团队成员卡片
- [ ] 当前用户卡片高亮显示
- [ ] 每个卡片显示成员头像、姓名、角色、更新数量
- [ ] 点击卡片内的日志可查看详情（仅自己的可操作）
- [ ] 提供"写工作日志"快捷按钮
- [ ] 支持查看不同月份的日志

**权限规则：**
- 可以查看所有人的日志基本信息
- 只能操作自己的日志（编辑、查看详情）
- 只有自己可以生成自己的月报

---

#### 2.2.3 撰写工作日志
**功能描述：** 结构化记录每日工作，支持工作事项、交付物、重点说明

**用户故事：**
> 作为团队成员，我想要快速记录今天的工作，以便月底自动生成月报。

**验收标准：**
- [ ] 可以选择工作日期
- [ ] 可以添加多个工作事项
- [ ] 每个事项可选择分类（预设分类）
- [ ] 每个事项可填写富文本描述
- [ ] 每个事项可上传交付物文件
- [ ] 支持添加重点说明
- [ ] 工作事项可以拖拽排序
- [ ] 支持保存草稿
- [ ] 发布后跳转到看板页
- [ ] 发布成功有 toast 提示

**工作分类（预设）：**
| 分类 | 说明 |
|------|------|
| 功能开发 | 新功能实现 |
| Bug修复 | 问题修复 |
| UI优化 | 界面与交互改进 |
| 代码重构 | 代码质量提升 |
| 性能优化 | 速度与体验优化 |
| 测试工作 | 测试相关 |
| 文档编写 | 文档与说明 |
| 需求沟通 | 需求评审、讨论 |
| 项目管理 | 项目协调与管理 |
| 技术分享 | 学习、分享 |

---

#### 2.2.4 个人月报
**功能描述：** 查看和管理个人月度工作汇总

**用户故事：**
> 作为团队成员，我想要查看自己的月度工作统计，以便了解自己的工作产出。

**验收标准：**
- [ ] 可以按月份选择查看
- [ ] 显示统计概览（日志数、交付物数、最活跃日期等）
- [ ] 显示工作分类分布
- [ ] 显示每日活动记录
- [ ] 列出详细的日志列表
- [ ] 支持导出月报（PDF/Markdown）
- [ ] 支持分享月报

**统计指标：**
| 指标 | 说明 |
|------|------|
| 总日志数 | 本月记录的日志条数 |
| 总交付物 | 上传的交付物数量 |
| 最活跃日期 | 日志最多的一天 |
| 工作分类分布 | 各类别工作占比 |
| 每日活动 | 全月每日的日志记录情况 |

---

#### 2.2.5 团队月报
**功能描述：** 查看团队整体的月度工作汇总

**用户故事：**
> 作为项目主管，我想要看到团队成员的工作情况汇总，以便进行项目管理和汇报。

**验收标准：**
- [ ] 横向展示团队成员的月报卡片
- [ ] 每个卡片显示成员统计数据
- [ ] 当前用户可以点击查看自己的详细月报
- [ ] 可以按月份切换查看
- [ ] 显示团队整体统计

**权限规则：**
- 可以看到所有成员的统计概览
- 只能查看自己的详细月报内容
- 其他成员的月报卡片显示但不可点击查看详情

## 3. 用户流程

### 3.1 主要流程

#### 流程1：日常记录工作
```
登录 → 看板页 → 点击"写工作日志"
    ↓
填写工作事项 → 上传交付物 → 点击"发布"
    ↓
返回看板，看到新日志
```

#### 流程2：查看个人月报
```
登录 → 导航到"个人月报"
    ↓
选择月份 → 查看统计和详情
    ↓
（可选）导出或分享
```

#### 流程3：查看团队月报
```
登录 → 导航到"团队月报"
    ↓
查看团队成员汇总
    ↓
（如果是自己）点击查看详细月报
```

## 4. 界面设计规范

### 4.1 视觉风格
- **主色调：** 翡翠绿（Emerald）
- **风格：** 简洁、专业、现代化
- **设计系统：** Shadcn UI
- **响应式：** 支持桌面端使用

### 4.2 导航结构
```
顶部导航栏
├─ Logo/标题
├─ 导航菜单 [看板 | 写日志 | 个人月报 | 团队月报]
└─ 用户信息（头像、姓名、退出）
```

### 4.3 文案规范
| 场景 | 文案示例 |
|------|---------|
| 按钮 | 写工作日志、发布、导出、分享 |
| 提示 | 正在发布...、发布成功！ |
| 空状态 | 暂无日志记录、本月还没有工作记录 |
| 错误 | 发布失败，请重试、网络错误 |

## 5. 数据规范

### 5.1 工作日志数据结构
```typescript
interface WorkLog {
  id: string;
  userId: string;
  userName: string;
  userEmail: string;
  dateTime: string;          // ISO 8601
  date: string;             // YYYY-MM-DD
  workItems: WorkItem[];
  notes?: string;
  tags: string[];
  content: string;
  deliverables: Deliverable[];
}

interface WorkItem {
  category: string;
  description: string;
  files: File[];
}
```

### 5.2 月报数据结构
```typescript
interface MonthlyReport {
  userId: string;
  year: number;
  month: number;
  totalLogs: number;
  totalDeliverables: number;
  mostActiveDay: string;
  topCategories: CategoryCount[];
  dailyActivity: DayCount[];
  logs: WorkLog[];
}
```

## 6. 非功能需求

### 6.1 性能要求
- 页面加载 < 2秒
- 日志发布响应 < 3秒
- 看板滚动流畅

### 6.2 可用性要求
- 页面布局清晰，操作直观
- 关键操作有确认反馈
- 出错时有明确提示

### 6.3 兼容性要求
- 主流现代浏览器（Chrome、Firefox、Safari、Edge）
- 桌面端优先（1024px 以上）
