# TraceFlow 技术架构文档

## 1. 技术栈总览

### 1.1 核心技术选型

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端框架 | React + TypeScript + Vite | 现代化前端开发 |
| UI 组件库 | Shadcn UI + Tailwind CSS | 简洁专业的组件库 |
| 富文本编辑 | TipTap | 强大的富文本编辑器 |
| 后端 | Hono (运行于 Supabase Edge Functions) | 轻量快速的服务端 |
| 数据存储 | Supabase KV Store | 键值对存储 |
| 认证 | Supabase Auth | 用户管理 |
| 部署 | Supabase Edge Functions | Serverless 部署 |

## 4. 核心功能实现

### 4.1 前端路由

**App.tsx 路由结构：**

| 路径 | 组件 | 说明 |
|------|------|------|
| /login | Login | 登录页 |
| / | WorkLogBoard | 看板页（首页） |
| /create | CreateWorkLog | 撰写日志 |
| /monthly-report | MonthlyReport | 个人月报 |
| /monthly-report/view | MonthlyReportView | 查看月报 |
| /team-monthly-report | TeamMonthlyReport | 团队月报 |

**路由保护：**
- 使用 `useEffect` 检查登录状态
- 未登录用户重定向到 /login
- 已登录用户可以访问所有页面

### 4.2 认证与用户管理

**认证流程：**
1. 用户在前端登录/注册
2. Supabase Auth 返回用户信息和 Access Token
3. Token 存储在 Supabase 客户端中
4. 后续请求自动携带 Token

**用户信息获取：**
```typescript
const { data: { user } } = await supabase.auth.getUser();
const { data: { session } } = await supabase.auth.getSession();
```

### 4.4 数据存储设计

**键值对存储结构：**

| Key 格式 | Value 类型 | 说明 |
|----------|-----------|------|
| `log:{uuid}` | WorkLog | 单条工作日志 |
| `report:{uuid}` | MonthlyReport | 月报记录 |
| `user:{uuid}` | UserConfig | 用户配置 |

**日志数据结构：**
```typescript
interface WorkLog {
  id: string;
  userId: string;
  userName: string;
  userEmail: string;
  dateTime: string;              // ISO 8601
  tags: string[];
  content: string;
  deliverables: Deliverable[];
  rawData?: any;                  // 原始提交数据
}
```

**KV 存储查询策略：**
- 获取所有日志：`getByPrefix('log:')`
- 过滤处理：在内存中进行用户、时间筛选

## 5. 关键技术实现

### 5.1 富文本编辑器
- 使用 TipTap 库
- 支持基础格式化（加粗、斜体等）
- 集成到工作事项描述

### 5.2 文件上传
- 前端处理文件预览
- 支持多种文件类型
- 数据结构包含文件名、类型、预览 URL

### 5.3 数据可视化
- 使用 Recharts 库
- 月度报告中的图表（分类分布、每日活动）

### 5.4 状态管理
- 使用 React 原生 `useState`、`useEffect`
- 全局状态通过 Context 或 URL 参数传递
- 用户认证状态由 Supabase 客户端管理
